Spring概述 #

  1. 一个轻量级的开源JavaEE框架

  2. Spring可以解决企业应用开发的复杂性

  3. Spring有两个核心部分：IOC 和 AOP

    （1）IOC：控制反转，把创建对象过程交给Spring进行管理

    目的：为了降低耦合度（读取xml配置文件创建对象调用方法）

    （2）AOP：面向切面，不修改源代码进行功能增强

  4. Spring特点

    （1）方便解耦，简化开发

    （2）Aop编程支持

    （3）方便程序测试

    （4）方便和其他框架整合

    （5）方便进行事物的操作

    （6）降低API开发难度

IOC容器 #

  1. IOC底层原理（xml解析，工厂模式，反射）

  2. IOC接口（BeanFactory）

    BeanFactory:IOC容器基本实现，是Spring内部的使用接口，

    不提供开发人员使用

    *加载获取配置文件时不会创建对象，在获取对象（使用时）才创建对象

    ApplicationContext:BeanFactory接口的子接口，提供更多的功能，一般开发人员使用

    *加载配置文件时，就会把配置文件中的对象创建

    java
    //1.加载Spring配置文件
ApplicationContext context = new ClassPathXmlApplicationContext("bean1.xml");

//2.获取配置文件的对象
Book book = context.getBean("book", Book.class);

  3. IOC操作Bean管理（指两个操作 Spring创建对象，Spring注入属性（赋值），基于xml）

    *两种实现方式

    （1）基于xml配置文件方式实现创建 #

    在配置文件中使用bean标签，标签里添加对应属性，就可以实现创建对象

    常用属性

    *id:唯一标识（不能填特殊符号）

    *class:类全路径

    *name:与id类似（可以写特殊符号）但一般不使用

    *创建对象时，默认执行无参构造器完成对象创建

    （1）基于xml配置文件方式实现注入属性（2种） #

    DI:依赖注入，就是注入属性

IOC操作Bean管理（FactoryBean） #

  1. Spring两种类型Bean,(普通Bean和工厂Bean)

  2. 普通Bean，在配置文件中定义的什么类型就返什么类型

  3. 工厂Bean，在配置文件中返回类型可以和定义时的类型不一致

    第一步 创建类，让该类作为工厂bean，实现接口FactoryBean

    实现接口中的方法返回不同的类型

    java
    public class MyBean implements FactoryBean<Course> {

    //定义返回bean
    @Override
    public Course getObject() throws Exception {
        Course course = new Course();
        course.setCname("物理");
        return course;
    }

    @Override
    public Class<?> getObjectType() {
        return null;
    }

    @Override
    public boolean isSingleton() {
        return FactoryBean.super.isSingleton();
    }
}
xml
<bean id="myBean" class="test.collectionType.FactoryBean.MyBean"></bean>

​

测试类

IOC操作Bean管理（bean生命周期）

  1. 生命周期

    （1）对象创建到对象销毁的过程

  2. bean生命周期（一共七步）

    （1）通过构造器创建bean实例（无参构造器）

    （2）为bean属性设置值和对其他bean的引用（调用set方法）

    （3）把bean实例传递bean后置处理器的方法

    （4）调用bean初始化的方法（需要进行配置）

    （5）把bean实例传递bean后置处理器的方法

    （6）bean可以使用（对象获取到了）

    （7）当容器关闭时，调用bean的销毁的方法（需要配置销毁方法）

  3. 周期显示实例

    java
    public class Orders {

    private String oname;

    public Orders() {
        System.out.println("无参构造");
    }

    public void setOname(String oname) {
        this.oname = oname;
        System.out.println("调用set方法赋值");
    }

    //创建初始化执行方法
    public void initMethod() {
        System.out.println("执行初始化方法");
    }

    //创建销毁方法
    public void destroyMethod() {
        System.out.println("执行销毁方法");
    }
}
xml
<bean id="orders" class="test.collectionType.bean.Orders" init-method="initMethod" destroy-method="destroyMethod">
    <property name="oname" value="手机"></property>
</bean>

IOC操作Bean管理（基于注解） #

  1. 什么是注解，代码里的特殊标记，格式：

    @注解名称(属性名称=属性值, 属性名称=属性值, ...)

​ 使用注解，注解作用在类上面，方法上面，属性上面

​ 使用注解目的：简化xml配置

  1. Spring针对Bean管理中创建对象提供注解

    1. @Component

    2. @Service

    3. @Controller

    4. @Repositroy

      *功能一样都可以创建对象，都可以来创建bean实例

  2. 基于注解方式创建对象

    1. 第一步，引入依赖（aop）

    2. 第二步，开启组件扫描（扫描注解）

      xml
      <!--开启组件扫描
扫描多个包
1.  base-package="包名1, 包名2, ...”中间用,隔开
2.  写扫描包的上层目录
-->
<context:component-scan base-package="test.dao, test.service"></context:component-scan>

    3. 第三步，创建类，在类上面添加创建对象的注解

    4. 设置扫描哪些和不扫描哪些

      xml
      <!--只扫描某个指定文件下带有指定标签注解的属性-->

</context:component-scan>

<!--扫描某个指定文件下所有属性不包括带指定标签注解的属性-->

                            expression="org.springframework.stereotype.Controller"/>
</context:component-scan>

    5. 基于注解方式实现属性注入

      1. @AutoWried (根据属性类型注入)

        第一步 将service和dao对象创建，在service和dao类上添加注解

        第二步 在service注入dao对象，在service类添加dao类型属性，在属性上面使用注解

      2. @Qualifier (根据属性名称注入)

        该注解要和@Autowired一起使用

        示例

        xml
        <context:component-scan base-package="test.dao, test.service"></context:component-scan>
        java
        @Repository(value = "userDaoImpl1")
public class UserDaoImpl implements UserDao{
    @Override
    public void add() {
        System.out.println("add");
    }
}
        java
        @Service
public class UserService {
    //定义属性
    @Autowired
    @Qualifier(value = "userDaoImpl1")

AOP #

面向切面编程：不通过修改源代码的方式对功能的增删

AOP底层原理

  1. 使用动态代理方式（两种情况）

    1. 有接口，JDK动态代理

      --创建一个接口，一个该接口是实现类，若想拓展接口的功能

      --则使用JDK动态代理，创建该接口代理对象

    2. 没有接口，CGLIB动态代理

      --创建一个类，增强该类功能但是没有接口

      --创建当前类的子类的代理对象

AOP（JDK动态代理）

  1. 使用JDK动态代理，使用Proxy类里面的方法创建代理对象

    1. 调用newProxyInstance方法

      参数1：类加载器

      参数2：增强方法所在的类，这个类实现的接口，支持多参数

      参数3：实现这个接口InvocationHandler,创建代理对象，写增强的方法

    2. 编写动态代理代码（例子） #

java
//接口
package com.BitterGourd02;

public interface SellItem {
    public void sell();
}
java
//实现接口
package com.BitterGourd02;

public class People implements SellItem{
    @Override
    public void sell() {
        System.out.println("售出物品");
    }
}
java
//代理类
package com.BitterGourd02;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

public class PeopleProxyClass implements InvocationHandler {

    //被代理的接口
    private SellItem sellItem;

    public void setSellItem(SellItem sellItem) {
        this.sellItem = sellItem;
    }

    //生成得到代理类
    public Object getProxy() {
        return Proxy.newProxyInstance(this.getClass().getClassLoader(), sellItem.getClass().getInterfaces(), this);
    }

    //处理代理实例，并返回结果
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        SellItem result  =  (SellItem) method.invoke(sellItem, args);
        System.out.println("执行业务完毕");
        return result;
    }
}
java
//调用
package com.BitterGourd02;

public class Client {
    public static void main(String[] args) {
        //真实的角色
        People people = new People();
        //代理角色，不存在
        PeopleProxyClass peopleProxyClass = new PeopleProxyClass();
        //设置代理的对象
        peopleProxyClass.setSellItem(people);
        //动态生成代理类
        SellItem proxy = (SellItem) peopleProxyClass.getProxy();
        proxy.sell();
    }
}

事务传播 #

隔离级别 #

  1. 脏读

    一个未提交的事务读到另一个未提交的事务的数据

  2. 不可重复读

    一个未提交的事务读到另一个提交事物修改数据

  1. 虚读

    一个未提交事务读取到另一个提交事务添加的数据

解决方式-设置事务隔离性 #

java
@Transaction(propagation = Propagation.REQUIRED, isolation = Isolation.REPEATABLE_READ)

事务参数 #

timeout：超时时间

​ 事务须在一定时间内进行提交，否则回滚

​ 默认值-1，设置时间以秒单位计算

readOnly：是否只读

​ 读：查询操作，写：添加修改删除操作

​ readOnly默认值false，表示可以查询，可以添加修改删除操作

rollbackFor：回滚

​ 设置出现哪些异常进行回滚

noRollbackFor：不回滚

​ 设置出现哪些异常不进行回滚
